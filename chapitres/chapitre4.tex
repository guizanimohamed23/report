\section{Introduction}

\justifying
This sprint focuses on the scanning pipeline and on the vulnerability processing features of the HexStrike platform. The goal of this iteration is to implement reliable scan orchestration, normalize heterogeneous tool outputs into a unified schema, match findings against the vulnerability catalog, and persist results for later analysis and reporting.

In practice, this sprint covers scan submission, queueing and scheduling, worker orchestration, normalization, vulnerability matching, result persistence, and the user interfaces to submit and view scans.

\section{Sprint Purpose}

\justifying
The purpose of this sprint is to build a reliable and scalable scanning infrastructure that transforms raw target submissions into actionable vulnerability intelligence. Concretely, the sprint delivers a robust queue and worker system for tool orchestration, a normalization pipeline that unifies outputs across heterogeneous security tools, a matching engine that correlates findings with known vulnerabilities, and a user interface for scan management and result analysis. This foundation enables efficient vulnerability detection and establishes the data pipeline required for advanced attack graph generation in subsequent sprints.

\begin{table}[H]
    \centering
    \caption{Use case description: Submit Scan Target}
    \label{tab:submit-scan}
    \renewcommand{\arraystretch}{1.4}
    \small
    \begin{tabularx}{\textwidth}{|p{2.5cm}|X|}
        \hline

        \textbf{Actors} & Authenticated User \\\hline
        \textbf{Description} & An authenticated user submits a target URL (and optional parameters); the system validates the input, creates a scan job, enqueues it, and returns a job identifier for tracking. \\\hline
        \textbf{Precondition} & User is authenticated and the submission endpoint and queue service are available. \\\hline
        \textbf{Nominal Scenario} & 1. User enters target URL and parameters. 2. System validates the target and policies. 3. System creates a scan record, enqueues the job, and returns a unique job ID. \\\hline

        \textbf{Logical Constraints} & Enforce URL validation, blacklist/whitelist policies, rate-limiting, and unique job identifiers. \\\hline
