\section{Introduction}

This sprint is dedicated to attack graph generation. The goal is to transform project artifacts into diagrams that show how data, endpoints and vulnerabilities are connected, so these diagrams highlight potential attack paths in the platform.

\section{Sprint Purpose}

The purpose of this sprint is to implement the Dashboard's attack-path and scenario generation features. The generator is an AI-assisted service that produces ephemeral attack scenarios (ordered steps) and a graph object (nodes and edges) for visualization in the Dashboard. Unlike static report diagrams, these graphs are runtime artifacts produced on-demand and returned to the frontend for interactive rendering.

\section{Design}
\subsection{Graph Data Model}

The runtime graph payload returned by the backend follows the shape used by the frontend React Flow renderer. Core elements:
\begin{itemize}
  \item \textbf{Step (source data)}: \texttt{\{ step\_id, title, description, mitre\_technique, severity, next\_steps: [number] \}} — produced by the AI or derived by the service.
  \item \textbf{Node}: \texttt{\{ id: string, data: \{ stepNumber, title, description, techniqueId, severity \}, position?: \{x,y\} \}} — represents a step in the graph.
  \item \textbf{Edge}: \texttt{\{ id: string, source: string, target: string, animated?: boolean \}} — connects nodes according to \texttt{next\_steps}.
\end{itemize}

Node attributes are intentionally minimal and frontend-friendly; additional metadata such as \texttt{aiRaw} and \texttt{recommendations} are returned alongside the graph for UI display.

\subsection{Sequence Diagram (runtime)}

Typical runtime sequence:
\begin{enumerate}
  \item Dashboard requests \texttt{POST /api/attacks/ai-generate} with \texttt{\{ vulnerabilityId, stepCount?, language? \}}.
  \item \texttt{attackController} validates input and calls \texttt{attackService.generateScenarioWithAI()}.
  \item \texttt{attackService} loads the \texttt{Vulnerability} (via Prisma), constructs the AI prompt, and sends it to the AI gateway (\texttt{OLLAMA\_BASE\_URL}).
  \item The AI gateway returns text; `attackService` extracts JSON, validates it, maps steps into nodes/edges, and returns the ephemeral payload.
  \item The controller returns the payload to the frontend; `attackApi.normalizeScenarioPayload()` ensures types/safety; the UI renders with React Flow.
\end{enumerate}

\section{Implementation}

The implementation centers on the backend \texttt{attackService} and \texttt{attackController}, and on the frontend normalization + React Flow renderer.

Backend details (source files):
\begin{itemize}
  \item \texttt{src/controllers/attackController.js} — exposes \texttt{generateAttackScenarioWithAI} and \texttt{generateRiskAssessmentWithAI} endpoints.
  \item \texttt{src/services/attackService.js} — contains core logic: input validation, DB retrieval (Prisma), AI prompt construction, AI request handling, JSON extraction and normalization (\texttt{mapAiOutputToScenarioPayload}, \texttt{mapAiOutputToEphemeralGraph}).
  \item Error handling covers invalid inputs, missing DB records, AI timeouts, and invalid AI outputs (returns 400/404/502/504 as appropriate).
\end{itemize}

Frontend details (source files):
\begin{itemize}
  \item \texttt{src/api/attackApi.ts} — \texttt{generateAttackScenario()} posts to \texttt{/api/attacks/ai-generate} and \texttt{normalizeScenarioPayload()} converts the response into typed \texttt{nodes} and \texttt{edges}.
  \item UI rendering uses React Flow where \texttt{nodes} and \texttt{edges} are passed to the flow component; nodes are positioned and displayed using a custom \texttt{AttackStep} node type.
\end{itemize}

Notes:
\begin{itemize}
  \item The graph produced is ephemeral (returned in the API response). The system may include \texttt{aiRaw} in the response for debugging, but production should avoid persisting raw AI outputs without review.
  \item The AI gateway must be reachable and configured via \texttt{OLLAMA\_BASE\_URL}, otherwise the service returns an error explaining the gateway issue.
\end{itemize}
